# Wine Quality 数据集模型评估与对比分析
## 模型评估
根据程序运行结果，对线性回归与逻辑回归模型在测试集上的表现评估如下：
### A. 线性回归模型（质量评分预测）
**评估指标**：均方误差 (MSE) = 0.3866
**结果分析**：
红酒质量评分范围为 0-10 分。MSE 为 0.3866，意味着预测值与真实值之间的平均偏差约为  $\sqrt{0.3866} \approx 0.62$  分。
**评价**：在不使用任何高级库的情况下，手写的梯度下降算法成功收敛。0.62 的平均误差说明模型能够较为准确地捕捉到化学成分（如酒精含量、酸度等）与评分之间的线性关系。
### B. 逻辑回归模型（好/坏酒分类）
**指标数据**：
准确率 (Accuracy) = 86.52% 
精确率 (Precision) = 59.09% 
召回率 (Recall) = 27.66%
**结果分析**：
准确率陷阱：
虽然 86.52% 的准确率看似很高，但结合召回率仅有 27.66% 可以发现，数据集中“好酒”（质量 > 6）的样本占比较少。
性能表现：
模型倾向于将红酒判定为“坏酒”（多为 5 或 6 分的中等酒），导致对“好酒”的捕捉能力较弱。这说明在类别不平衡的情况下，单纯依靠准确率无法全面衡量模型优劣，模型在处理少数类样本（好酒）时仍有提升空间。

## 模型异同点分析

虽然两个模型都利用了特征的线性组合，但在本质上存在显著差异：
A. 相同点
**基本假设**：两者都假设输出（或输出的某种形式）与输入特征  $X$  之间存在线性关系，核心计算公式均为  $z = w^T X + b$ 。
**优化手段**：本实验中两者均采用梯度下降法 (Gradient Descent) 进行参数更新，其权重更新的代数形式在数学推导后具有一致性。
**预处理依赖**：两者对特征尺度都非常敏感，本实验中通过手动实现 Z-score 标准化 才确保了模型的收敛。

## 模型异同点分析

| 比较维度 | 线性回归 (Linear Regression) | 逻辑回归 (Logistic Regression) |
| :--- | :--- | :--- |
| **任务目标** | **回归**：预测红酒具体质量得分 | **分类**：区分好酒 (Quality > 6) 与坏酒 |
| **输出形式** | 连续实数值 $(-\infty, +\infty)$ | 概率值，范围在 $(0, 1)$ 之间 |
| **激活函数** | 无 | **Sigmoid 函数** |
| **损失函数** | **均方误差 (MSE)** | **二元交叉熵损失 (Log-Loss)** |
| **评估指标** | 均方误差 (MSE) | 准确率 (Accuracy)、召回率 (Recall) |
